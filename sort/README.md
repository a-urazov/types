# Сортировка

Пакет `sort` предоставляет эффективные и оптимизированные алгоритмы сортировки для срезов (slices) элементов любого типа. Реализация использует гибридный подход с параллельной сортировкой для больших наборов данных.

## Основные функции

- `Slice[T](s []T, less func(a, b T) bool)` - сортирует срез в соответствии с заданной функцией сравнения

## Пример использования

```go
package main

import (
    "fmt"
    "types/sort"
)

func main() {
    // Сортировка целых чисел
    numbers := []int{5, 2, 8, 1, 9}
    sort.Slice(numbers, func(a, b int) bool {
        return a < b
    })
    fmt.Println("Sorted numbers:", numbers) // [1 2 5 8 9]

    // Сортировка строк по длине
    words := []string{"zebra", "apple", "go", "database"}
    sort.Slice(words, func(a, b string) bool {
        return len(a) < len(b)
    })
    fmt.Println("Sorted by length:", words) // [go apple zebra database]

    // Сортировка пользовательских типов
    type Person struct {
        Name string
        Age  int
    }

    people := []Person{
        {"Alice", 30},
        {"Bob", 25},
        {"Charlie", 35},
    }
    sort.Slice(people, func(a, b Person) bool {
        return a.Age < b.Age
    })
    for _, p := range people {
        fmt.Printf("%s: %d\n", p.Name, p.Age)
    }
}
```

## Особенности реализации

- **Гибридная сортировка**: комбинирует быструю сортировку с сортировкой вставками
- **Параллельная сортировка**: использует несколько горутин для больших массивов
- **Оптимизированная кэш производительность**: пороговое значение 16 элементов для переключения на сортировку вставками
- **Адаптивная**: настраивает количество горутин на основе GOMAXPROCS

## Параметры оптимизации

- **MaxGoroutines**: максимальное количество горутин для параллельной сортировки (по умолчанию: GOMAXPROCS)
- **Threshold**: размер массива, при котором переключается на сортировку вставками (по умолчанию: 16)

## Производительность

| Случай  | Временная сложность |
| ------- | ------------------- |
| Среднее | O(n log n)          |
| Лучшее  | O(n log n)          |
| Худшее  | O(n²) (редко)       |

## Применение

- Сортировка больших наборов данных
- Сортировка в производительно-критичных операциях
- Сортировка с пользовательским порядком сравнения
- Параллельная обработка больших массивов
