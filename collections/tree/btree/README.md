# B-Tree

Пакет `btree` предоставляет реализацию B-дерева. B-дерево - это многоуровневое дерево поиска, которое оптимизировано для систем с блочным доступом к памяти (как в базах данных и файловых системах).

## Основные методы

- `New[T](degree int) *Tree[T]` - создает новое B-дерево со степенью ветвления
- `Insert(value T)` - вставляет значение в дерево
- `Search(value T) bool` - ищет значение в дереве
- `Delete(value T) bool` - удаляет значение из дерева
- `Range(min, max T, fn func(T))` - обходит все значения в диапазоне [min, max]

## Пример использования

```go
package main

import (
    "fmt"
    "types/collections/tree/btree"
)

func main() {
    tree := btree.New[int](3) // B-Tree с максимум 3 ключами на узел
    
    for i := 1; i <= 10; i++ {
        tree.Insert(i)
    }
    
    if tree.Search(5) {
        fmt.Println("5 found!")
    }
    
    // Обход значений в диапазоне
    tree.Range(3, 7, func(val int) {
        fmt.Println(val)
    })
}
```

## Особенности

- **Сбалансированное**: дерево остается сбалансированным после каждой операции
- **Степень ветвления**: узел может содержать от ceil(t/2) до t ключей
- **Оптимизировано для блочного доступа**: идеально для баз данных и файловых систем
- **Гарантированная высота**: O(log n)

## Производительность

| Операция | Временная сложность |
|----------|---|
| Поиск | O(log n) |
| Вставка | O(log n) |
| Удаление | O(log n) |
| Диапазонный поиск | O(log n + k), где k - количество результатов |

## Степень (Order) B-дерева

Степень определяет максимальное количество дочерних узлов:
- B-Tree степени 3: до 2 ключей на узел, до 3 дочерних узлов
- B-Tree степени 4: до 3 ключей на узел, до 4 дочерних узлов

## Применение

- Базы данных (индексирование)
- Файловые системы
- Поиск в больших последовательных наборах данных
- Системы, где доступ к диску - основное узкое место
- Многоуровневое индексирование
