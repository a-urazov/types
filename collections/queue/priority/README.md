# Priority Queue

Пакет `priority` предоставляет реализацию приоритетной очереди. В отличие от обычной очереди, где элементы извлекаются в порядке FIFO, приоритетная очередь извлекает элементы в порядке их приоритета.

## Основные методы

- `New[TElement, TPriority]() *Queue[TElement, TPriority]` - создает новую пустую приоритетную очередь
- `Enqueue(element TElement, priority TPriority)` - добавляет элемент с заданным приоритетом
- `Dequeue() (TElement, bool)` - удаляет и возвращает элемент с наивысшим приоритетом
- `Peek() (TElement, bool)` - получает элемент с наивысшим приоритетом без удаления
- `Size() int` - возвращает количество элементов
- `IsEmpty() bool` - проверяет, пуста ли очередь
- `Clear()` - очищает очередь

## Пример использования

```go
package main

import (
    "fmt"
    "types/collections/queue/priority"
)

func main() {
    pq := priority.New[string, int]()
    
    pq.Enqueue("low priority task", 1)
    pq.Enqueue("high priority task", 10)
    pq.Enqueue("medium priority task", 5)
    
    // Извлекаются в порядке: высокий приоритет сначала
    for !pq.IsEmpty() {
        if item, ok := pq.Dequeue(); ok {
            fmt.Println("Processing:", item)
        }
    }
}
```

## Особенности

- **Приоритизация**: элементы извлекаются на основе приоритета
- **Эффективность**: использует внутреннюю кучу (heap) для O(log n) операций
- **Потокобезопасность**: встроенная синхронизация
- **Гибкость**: поддерживает любые сравнимые типы приоритетов

## Применение

- Планировщики операционной системы
- Алгоритм Dijkstra для поиска кратчайшего пути
- Huffman кодирование
- Системы обработки событий с приоритетами
- Системы очереди задач
