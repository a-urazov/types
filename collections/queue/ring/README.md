# Ring Buffer

Пакет `ring` предоставляет реализацию кольцевого буфера (Ring Buffer) - структуры данных фиксированного размера, которая переиспользует пространство, перезаписывая старые элементы, когда буфер переполняется.

## Основные методы

- `New[T](capacity int) *Buffer[T]` - создает новый кольцевой буфер с заданной вместимостью
- `Write(item T)` - добавляет элемент в буфер (перезаписывает старый, если буфер полон)
- `Read() (T, bool)` - получает следующий элемент из буфера
- `Capacity() int` - возвращает максимальный размер буфера
- `Size() int` - возвращает текущее количество элементов
- `IsFull() bool` - проверяет, полон ли буфер
- `IsEmpty() bool` - проверяет, пуст ли буфер
- `Clear()` - очищает буфер

## Пример использования

```go
package main

import (
    "fmt"
    "types/collections/queue/ring"
)

func main() {
    rb := ring.New[int](5)
    
    // Добавляем элементы
    rb.Write(1)
    rb.Write(2)
    rb.Write(3)
    rb.Write(4)
    rb.Write(5)
    rb.Write(6) // Перезапишет 1
    
    // Читаем элементы
    for !rb.IsEmpty() {
        if item, ok := rb.Read(); ok {
            fmt.Println("Read:", item)
        }
    }
}
```

## Особенности

- **Фиксированный размер**: вместимость определяется при создании
- **Циклическая структура**: переиспользует пространство после конца
- **Потокобезопасность**: встроенная синхронизация
- **Эффективность памяти**: не требует переустройства массива при добавлении элементов

## Применение

- Буферизация аудио/видео потоков
- Логирование событий с ограниченным буфером
- Скользящие окна в обработке сигналов
- Кэширование последних N значений
- Системы реального времени с фиксированной памятью
